## stm32基本认识

单片机就是微型计算机，所有计算机需要的功能都集成在一块芯片上了

位数指的是CPU一次处理的数据位宽，位宽越高，性能越强

![1763464068047](image/嵌入式开发/1763464068047.png)

ST官方有着一套命名规则，要根据具体的运用选用不同的芯片系列

![1763464227726](image/嵌入式开发/1763464227726.png)

stm32芯片内的总线一共11条，主控总线有S0\S1\S2\S3\S4\S5，被控总线M0\M1\M2\M3\M4

![1763464575393](image/嵌入式开发/1763464575393.png)

stm32使用C语言编程的本质，是在操作各种**外设寄存器**

stm32有4GB的地址空间，st公司已经大致规定死了地址分配情况

![1763465503383](image/嵌入式开发/1763465503383.png)

如果寄存器很少的话，可以直接查地址操作寄存器地址编程，但是stm32地址太多了，所以st公司已经将各种地址操作都封装成了函数，也就是我们所说的标准库和HAL库

#### 推挽输出

该模式下，在输出驱动器中，P-MOS和N-MOS协同工作

假设小灯是高电平工作，程序将数据传入寄存器，然后输出驱动器将P-MOS连通，N-MOS关闭，这样就会使得输出高电平到外部电路，I/O引脚接收高电平，小灯点亮

![1749048831002](image/嵌入式开发/1749048831002.png)

推挽输出只能实现3.3V的高电平输出，如果模块需要更大的电平进行驱动，就无法使用推挽输出

#### 开漏输出

开漏输出有更大的电压工作范围

开漏输出模式下，输出驱动器中只有N-MOS在工作，但是工作模块并不接地，而是存在外接电源，例如这里小灯原本的接地端直接接入了5V的高电平，只要N-MOS连通，小灯就能变为通路，有外接5V电源提供电压

![1749049306501](image/嵌入式开发/1749049306501.png)

值得注意的是，因为外接电源的缘故，模块的I/O口不能随意选区，要根据外接电压大小选择能承受相应高电压的I/O口，避免造成损坏

开漏输出依赖外部电源提供驱动，更加灵活

推挽输出直接提供内部电源进行驱动

![1749049650054](image/嵌入式开发/1749049650054.png)

#### 复用推挽输出

复用推挽输出接收的指令来自片上外设，原理与推挽输出一致

#### 复用开漏输出

复用开漏输出接收的指令来自片上外设，原理与开漏输出一致

#### 浮空输出

相对于接口处连接了一个无限大的电阻，在不按下按键时，是高电平，按下按键后会接地，因此是低电平

### 使用标准库进行点灯

标准库的特点就是操作寄存器，所以我们需要查阅手册了解GPIO的寄存器地址

GPIO的初始化流程（以GPIOB8为例）

* 使能GPIO时钟
* 配置GPIOB的第8个IO口
* 配置PB8为输出模式
* 配置PB8的输出类型为推挽
* 配置PB8的输出速度为100MHz
* 初始化GPIOB

```c
#include "stm32f4xx.h"

void delay(uint32_t);

int main()
{
	GPIO_InitTypeDef GPIO_InitStructer;
	RCC_AHB1PeriphClockCmd(RCC_AHB1Periph_GPIOB,ENABLE);

	GPIO_InitStructer.GPIO_Mode=GPIO_Mode_OUT;
	GPIO_InitStructer.GPIO_OType=GPIO_OType_PP;
	GPIO_InitStructer.GPIO_Pin=GPIO_Pin_8;
	GPIO_InitStructer.GPIO_PuPd=GPIO_PuPd_UP;
	GPIO_InitStructer.GPIO_Speed=GPIO_High_Speed;

	GPIO_Init(GPIOB,&GPIO_InitStructer);
	while(1)
	{
		GPIO_SetBits(GPIOB, GPIO_Pin_8);
		delay(200);
		GPIO_ResetBits(GPIOB, GPIO_Pin_8);
		delay(200);
	}
}

void delay(uint32_t cnt)
{
	uint32_t i,j;
	for(i=0;i<cnt;i++)
	{
		for(j=0;j<0xFFF;j++)
		{}
	}
}
```

## 中断

### 中断机制

计算机中异常的一种，嵌入式设备也是通用的

正在执行任务1时，中断任务1，先处理更重要的任务2，任务2执行完成后再执行任务1

![1750299185313](image/嵌入式开发/1750299185313.png)

触发中断的事件有很多：指令出错、计时器时间结束、串口接受到数据、GPIO口电平发生变化

EXIT0到EXIT15对应的是16根信号线，可以将对应的GPIO按钮设置为输出到EXIT信号线

中断的过程（stm32电路）：输入驱动器处理GPIO输入信号——>边沿检测电路——>请求挂起寄存器——>NVIC(中断控制器)——>NVIC根据对应中断向量执行对应中断处理函数——>清零请求挂起寄存器，防止重复触发

![1750301219649](image/嵌入式开发/1750301219649.png)

![1750300742288](image/嵌入式开发/1750300742288.png)

![1750300788350](image/嵌入式开发/1750300788350.png)

![1750300846281](image/嵌入式开发/1750300846281.png)

多个中断同时触发时，可以设置优先级来规定执行顺序

优先级分为抢占优先级和响应优先级

多个中断同时触发时，先比较抢占优先级再比较响应优先级，数字越小越优先

### 串口原理

#### 串口发送

stm32中有着与发送相关的寄存器有发送数据寄存器（TDR）、发送移位寄存器

发送移位寄存器将数据转换成高低电平从TX引脚输出

**发送数据寄存器**存储下一位要发送的数据，stm32的CPU会不断查询发送寄存器是否发送完，如果发送完了，就会将**发送数据寄存器TDR**中的数据传给**发送移位寄存器**，然后把下一位待发送数据传到**发送数据寄存器TDR**中

![1750384866844](image/嵌入式开发/1750384866844.png)

#### 串口接收

stm32中有着与发送相关的寄存器有**接收数据寄存器（RDR）**、**接收移位寄存器**

**接收移位寄存器**读取RX传来的高低电平信号，每读取八位，就传给**接收数据寄存器（RDR）**，CPU读取RDR中有一位数据，就将数据读入储存器

### 中断模式收发

#### 阻塞现象

在轮询模式收发过程中，CPU会一直查询数据寄存器，不管是接收还是发送，使得cpu无法完成其他任务，这种情况称为“阻塞现象”

要消除阻塞，可以使用中断机制，当移位寄存器发送完了，CPU再去完成数据转移操作，其他时间可以进行其他任务

#### 中断收发函数

中断接收函数HAL_UART_Receive_IT有三个参数

* 第一个是需要操作串口的指针
* 第二个参数是要发送数据的指针
* 第三个参数是发送数据的长度
* 因为不需要占用cpu使得阻塞,所以中断接收函数并没有第四个参数超时时间

HAL_UART_Receive_IT不会阻塞程序,写在循环体内会导致程序还没接收完数据就会继续向下执行了,因此函数要写在循环体外

## I2C

### I2C通信原理

I2C通信和TTL串口一样也是使用两根线，一条共地线进行通信

两根数据线中，一根用于数据传输SDA，另一根用于传输时钟频率SCL

![1750470985967](image/嵌入式开发/1750470985967.png)

I2C通信是

* 半双工通信：同一时间只能接收或发送，不能同时进行收发
* 主从模式：只要主机发起通信请求，从机才能开始通信，因此I2C能够同时连接多个设备
* 总线协议：能同时对多个设备进行通信的协议
* 同步通信：由主机发送固定的频率信号作为总线上的统一时间

I2C通信协议中，每个设备都有自己唯一的地址，当需要与某个从机通信时，就可以发送对应的地址进行通信

I2C通信过程：需要发送信号时，主机先在时钟线高电平时，拉低通信电平作为通信开始的信号，当主机发送完数据后，在低电平时由从机将电平拉低一次作为应答信号，随后总线控制权转交给从机，从机发送完信号后，主机在时钟处于高电平时将数据线拉高，结束通信

只有在发送开始通信和结束通信时，主机才会在时钟线高电平进行操作，其余时间的通信都是在时钟线低电平发送数据的

### I2C中断、DMA与状态机编程

#### 中断方式

不使用中断机制的话，I2C正常收发过程中CPU会一直被占用。无法完成其他任务

通信时，每发送一个字节的数据从机都要发送应答信号

中断模式能做到当从机发送应答信号之后，CPU会来处理一字节的数据发送，然后再去做其他任务，直到再次收到从机的应答信号

DMA模式是将所有的字节数据发送都交给DMA来完成，直到所有数据发送完再通过中断通知CPU来处理

在I2C1的NVIC配置中，将I2C1的事件中断和错误中断都打开，主要使用事件中断

{
	uint8_t sendBuffer[3]={0xAC,0x33,0x00};
	uint8_t readBuffer[6]={0};

	HAL_I2C_Master_Transmit_IT(&hi2c1, AHT_ADDRESS,sendBuffer, 3);
	HAL_Delay(75);
	HAL_I2C_Master_Receive_IT(&hi2c1, AHT_ADDRESS, readBuffer, 6);

	if((readBuffer[0] & 0x80)==0x00){

		*Humidity=data*100.0f/(1<<20);

**因为使用了中断完成数据读取,所以程序在调用完读取函数后,就继续执行剩下的代码,不会等待数据读取完成,这就会导致,当运行到发送读取数据的代码时,数据还没有测量完，导致显示结果出错**

* 第二个参数是从机地址
* 第三个参数是接收数据的变量指针

## OLED

OLED屏幕的原理就是将很多的小灯进行排列,只要点亮需要的小灯就可以显示图像了

使用的是128列X64行的小灯

与使用GPIO口正常点亮小灯不同,一般的屏幕是使用屏幕驱动芯片来实现的(常用的有CH1116、SSD1306芯片)

![1751267890086](image/嵌入式开发/1751267890086.png)

芯片会将屏幕的小灯进行分页分组处理

将64行分为8页（每一页有8行），由0-127对每一列进行编号

![1751267787894](image/嵌入式开发/1751267787894.png)

这样分完后指定页号上每一列都有8个小灯，就更好可以使用1字节数据控制8个小灯的状态（一字节是八位）

### CH1116芯片通信

#### 指令通信

![1751268229928](image/嵌入式开发/1751268229928.png)

指令格式是三位

* 第一位是I2C接口的地址
* 第二位固定是0x00
* 第三位是指令

设置页地址

如果要设置page0的地址，则指令内容为0xB0，如果要设置page7的地址，则指令内容为0xB7

设置列地址

列地址设置要分两次发送，设置第90列的地址，先将低4为设置为A（指令为0x0A），再将高4位设置为5（指令为0x15）

#### 数据通信

![1751268816534](image/嵌入式开发/1751268816534.png)

* 第一位是I2C接口的地址
* 第二位固定是0x40
* 第3位往后，每个字节代表一列（8个）小灯的亮灭

CH1116芯片还有一个特性，就是每次设置完一位数据后，列地址会自动+1

例如设置完要操作第0页，第0列的小灯，接着发送数据，第三位会控制0页0列的小灯、第四位会控制0页1列的小灯......

**用法**:只要设置要操作第0页，第0列的小灯,然后使用数据发送的指令,发送128位数据,就可以设置完第0页小灯的亮灭,然后再让设置为第1页,继续循环8次就可以完成整个屏幕的亮灭了

### 基本定时功能

***定时器就是计数器***

定时器功能：计数、捕获脉冲宽度、计算PWM占空比、输出PWM波形、编码器计数

f10系列有8个定时器：TIM1~TIM8

按功能强弱分为

* 基本定时器：TIM6、TIM7
* 通用定时器：TIM2、TIM3、、TIM4、TIM5
* 高级定时器：TIM1、TIM8

计时原理：一串脉冲信号以恒定频率输入到寄存器，寄存器每收到一个上升沿就+1，那么只要读取寄存器中的数据，就可以得到计时时间

![1751341541824](image/嵌入式开发/1751341541824.png)

由于寄存器最多存放16bit位数据，如果输入72MHz的话，最大才能记录0.09s，不够用

为了解决这个问题，寄存器前添加了预分频器，当接收到了n个脉冲后就发送一个脉冲给计数器

预分频器本质上也是个计数器，要设置为n分频，那么分频器的数字要设置为n-1

![1751341750294](image/嵌入式开发/1751341750294.png)

定时器之间也是可以级联的，这样可以增加最大计时时间

自动重装载寄存器：定时m个脉冲，当计数器的值达到m-1时，就会将计数器清零，并且“提醒一次”（调用一次函数）

![1751342124626](image/嵌入式开发/1751342124626.png)

预分频器和自动重装载寄存器都有影子寄存器的机制，就是真正工作的其实是预分频器的拷贝，还有自动重装寄存器的拷贝

自动重装寄存器的影子寄存器机制，是可以手动打开的

![1751418924159](image/嵌入式开发/1751418924159.png)

设置预分频器和自动重装载寄存器，会在下一个计数周期才生效

补充：计数器三种模式

* 向上计数模式：从0开始计数到自动重装载预设值
* 向下计数模式：从预设值开始计数到0
* 中央对齐模式：从0开始计数到自动重装载预设值再倒数回0

### 定时器输出比较模式

定时器输出比较模式下,捕获寄存器被用作比较寄存器

![1751635269286](image/嵌入式开发/1751635269286.png)

输出比较模式下,比较寄存器设定值会一直与计数器里的值进行比较,根据比较结果判断输出高电平或者低电平

输出比较有四种模式

* 冻结模式:不输出,或者说比较结果不改变比较寄存器的输出
* 强制有效:无论比较结果如何,都输出有效电平
* 强制无效:无论比较结果如何,都输出无效电平
* 匹配模式:有三种设置,一是设置为与计数器相等时,输出有效电平,二是相等时输出无效电平,三是相等时翻转电平
* **PWM模式1**：计数器值小于比较寄存器时输出有效，计数器值大于比较寄存器时输出无效电平
* **PWM模式2**：计数器值小于比较寄存器时输出无效，计数器值大于比较寄存器时输出无效电平

![1751769481246](image/嵌入式开发/1751769481246.png)

比较寄存器的出口处存在一个输出控制器，可以设置输出何种电平，默认情况下寄存器输出有效电平时，GPIO口输出高电平，通过设置输出控制器可以翻转这种效果

```c
for(int i=0;i<100;i++){
		__HAL_TIM_SET_COMPARE(&htim3,TIM_CHANNEL_1,i);
		HAL_Delay(10);
	  }
	  for(int i=99;i>=0;i--){
	  	__HAL_TIM_SET_COMPARE(&htim3,TIM_CHANNEL_1,i);
	  	HAL_Delay(10);
	  }
```

**代码实现的PWM波形变化**

**第一个for循环（变亮过程）：**

* 比较值从0递增到99
* 占空比从0%逐渐增加到99%
* PWM波形的高电平时间逐渐变长
* LED亮度从熄灭逐渐变亮到接近最亮

**第二个for循环（变暗过程）：**

* 比较值从99递减到0
* 占空比从99%逐渐减少到0%
* PWM波形的高电平时间逐渐变短
* LED亮度从接近最亮逐渐变暗到熄灭

**代码实现的时间特性**

 每个亮度级别持续10ms（HAL_Delay(10)）

* 完整的变亮过程：100个步骤 × 10ms = 1秒
* 完整的变暗过程：100个步骤 × 10ms = 1秒
* 一个完整的呼吸周期：2秒

## ADC

### ADC原理

ADC是使用模拟信号输出数字信号的技术，有很多种实现方法

stm32使用的是逐次逼近法（SAR），原理就是二分比较

stm32f103使用的是12位ADC，也就是输出电压值由12位二进制数确定（0~4095），12位也被称为ADC的分辨率

比如0V对应二进制数0000 0000 0000，3.3V对应1111 1111 1111，因为stm32最大电压是3.3V，所以最大值（12位全是1）对应电压是3.3V

![1751952117155](image/嵌入式开发/1751952117155.png)

假如输入电压是0.9V，那么单片机会将0.9V保存在一个电容上，然后用1000 0000 0000（3.3V的一般1.65V）与之比较，如果待测电压比1.65V大就设置第12位为1，然后1再比较下一位二分位（0100 0000 0000）

依次循环往复，直到比较完12位数，就可以得到待测电压对应的二进制编码

然后会将该数值存入一个数据寄存器中，便于取用

比如测量结果转换为十进制是1116，那么用**1116/4095x3.3V**就可以得到待测电压

### ADC基本使用

原理只需了解即可，实际使用时，只需要这三部就可以使用

1启动ADC->2采样转换->3获取结果计算

stm32F103C8T6上的GPIO口有16个ADC采样通道

内部有两个ADC转换结构ADC1、ADC2，每一个结构内分规则组和注入组，规则组可以设定注册那几个ADC输入通道，输入电压后ADC结构就会将该通道的电压值放入**规则通道寄存器中**1

### ADC多通道采集

ADC1的规则组中存在一个注册表,可以注册多个通道,使得单个ADC可以读取多个通道的输入,每个通道数据转换完成后,会将数据传送到**规则通道寄存器(ADC存放数据的寄存器)**,并且将转换结束标志位置为1

![1751956600818](image/嵌入式开发/1751956600818.png)

开启ADC的**扫描模式**后,ADC会循环采集注册表上的所有通道,然后依次放到寄存器

但是可能会出现寄存器值覆盖的情况,上一个通道的数据还没从寄存器里取出,就被下一个通道的测量值覆盖了

解决这种情况可以使用DMA功能,设定一个数组变量,让DMA在ADC每次转换完成后,都将数据从寄存器搬运到数组内

# Vivado HLS设计

## HLS工作机制

CAD（计算机辅助设计）→CAE（计算机辅助工程）→EDA（电子设计自动化）→ESL（电子系统级的设计方法）

ESL

* 使用高级语言（C、C++）：Vivado HLS
* 基于模型的可视化设计工具：Xlinx的System Generator

Vivado HLS可以将c/c++转化为HDL的过程：scheduling（调度）→control logic extraction（控制逻辑提取）→binding（硬件资源映射）

HDL是硬件描述语言，主要有verilog和VHDL两种

## HLS设计流程

传统流程：HDL（verilog等）→RTL simulation（行为级仿真，验证算法）→synthesis（综合）→implementation（布局布线）→system debug

HLS设计流程：高级语言（C/C++）→C simulation（c语言仿真，验证算法）→c synthesis（c综合）→RTL Synthesis implementation（综合与布局布线）→system debug

综合时，会有三个指标：Latency、Timing、Resource

test bench是一个虚拟硬件环境，vivado中我们需要使用c/c++去描述一个环境，以此来在虚拟环境中验证我们的设计

stimulus提供输入数据

Reference Model：产生参考值

DUT：待测模型，我们需要综合的代码，

Scoreboard：把DUT结果与Reference进行比较

例子：测试一个加法程序
